# Agent Guide: Provider Model Directory

This directory is the cross-provider index for model pages that live under `providers/<provider>/models/`.

## Rules

- Treat provider-specific model pages as authoritative; this directory links and summarizes them.
- Use `[provider-doc]` for model availability, context windows, modalities, pricing, and tool support.
- Keep preview, deprecated, product-only, and API-callable models clearly separated.
- Use tables as the default comparison format. Avoid long paragraph-only model listings when rows and columns would make differences easier to scan.
- Include benchmark tables where applicable, but only for named benchmarks with a date, source, and methodology link. Mark whether scores are provider-reported or independently reported.
- Do not turn this into a benchmark ranking. If a benchmark is cited, name the benchmark, date, and methodology source.
