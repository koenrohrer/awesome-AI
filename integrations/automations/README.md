# Automations

Recurring or trigger-based AI workflows that do useful work without opening a fresh chat every time. This section is intentionally cross-surface: if the main story is the automation itself rather than the host platform, it belongs here.

## What belongs here

- Scheduled and event-driven routines
- Background agent jobs that triage, summarize, follow up, or ship work
- Shareable recipes that have proven useful in real workflows
- Cross-tool automations where the value is in the end-to-end routine, not just the platform

## Inclusion bar

`[tested]` preferred — especially for recipes that show inputs, trigger conditions, and outcomes.

`[vetted-tool]` is acceptable when the entry is a productized automation surface with a narrow, proven scope.

## What belongs elsewhere

- Shell-first automations with minimal orchestration detail — `integrations/shell/`
- Pipeline-native automation inside CI systems — `integrations/ci-cd/`
- No-code platform-first flows — `integrations/no-code/`
- First-party product docs for a provider surface — `providers/`

## Status

Introduced in v0.4. No curated automation entries yet.
